# Firebase Setup Guide — HR Workforce Management System

---

## Step 1 — Create Firebase Project

1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. Click **Add project**
3. Name it `HR-Workforce-App`
4. Disable Google Analytics (not needed) → **Create project**

---

## Step 2 — Enable Authentication

1. In your project sidebar → **Authentication** → **Get started**
2. Under **Sign-in method** → **Email/Password** → **Enable** → **Save**

---

## Step 3 — Enable Firestore

1. In your project sidebar → **Firestore Database** → **Create database**
2. Select **Start in test mode** (you can add security rules later)
3. Choose a region close to you → **Done**

---

## Step 4 — Install FlutterFire CLI & Configure

Run in any terminal (once globally):
```powershell
dart pub global activate flutterfire_cli
```

Then inside the Flutter project folder:
```powershell
cd "HR_Workforce_Management_System"
flutterfire configure
```

- Select your `HR-Workforce-App` project
- Select `android` (and `ios` if needed)
- This auto-generates `lib/firebase_options.dart` — **do not edit this file**

---

## Step 5 — Get Flutter Packages

```powershell
flutter pub get
```

---

## Step 6 — Create the First Admin User

### In Firebase Console → Authentication → Users → Add user:
| Field    | Value                      |
|----------|----------------------------|
| Email    | `admin@hrapp.internal`     |
| Password | *(your choice)*            |

> ⚠️ Always use the format `username@hrapp.internal` — this is what the app converts to internally when the user types their username at login.

> Copy the **UID** shown — you will need it in the next step.

### In Firestore → `users` collection → Add document:

- **Document ID**: *(paste the UID from Authentication)*
- **Fields**:

| Field          | Type    | Value                   |
|----------------|---------|-------------------------|
| `name`         | string  | `Admin`                 |
| `username`     | string  | `admin`                 |
| `email`        | string  | `admin@hrapp.internal`  |
| `role`         | string  | `admin`                 |
| `phone`        | string  | *(optional)*            |
| `department`   | string  | `Administrator`         |
| `isActive`     | boolean | `true`                  |
| `joinDate`     | string  | `2026-01-01`            |

```dart
import 'package:myapp/core/services/hr_rules_service.dart';
```

## Step 8 — Add Employee Users

For every employee, repeat **Step 6**:

1. **Authentication** → Add user:
   - Email: `username@hrapp.internal` (e.g. `john@hrapp.internal`)
   - Password: *(set a default, employee can change later)*

   > The username the employee uses to log in must match the prefix before `@hrapp.internal`.

2. **Firestore** → `users` collection → Add document with UID as ID:

| Field        | Value                          |
|--------------|--------------------------------|
| `name`       | Employee full name             |
| `username`   | Same prefix used in email      |
| `email`      | `username@hrapp.internal`      |
| `role`       | `employee`                     |
| `department` | e.g. `Developer`               |
| `phone`      | *(optional)*                   |
| `isActive`   | `true`                         |
| `joinDate`   | `YYYY-MM-DD`                   |

## Firestore Collections Reference

### `users`
Stores user profiles. Document ID = Firebase Auth UID.

Recommended `department` values:
- `Administrator`
- `Developer`
- `CRM`
- `Project Manager`
- `Trainee`

```
users/
  {uid}/
    name:         string
    username:     string        ← what the user types to log in
    email:        string        ← username@hrapp.internal
    role:         string        ← "admin" | "employee"
    phone:        string?
    department:   string?
    designationId: string?
    isActive:     boolean
    joinDate:     string        ← YYYY-MM-DD
```

---

### `attendance`
One document per employee per day.

```
attendance/
  {docId}/
    userId:       string        ← UID
    date:         string        ← YYYY-MM-DD
    punchIn:      timestamp
    punchOut:     timestamp?
    totalHours:   number
    isLate:       boolean
    lateMinutes:  number
    isHalfDay:    boolean
    status:       string        ← "present" | "absent" | "half-day"
```

---

### `leaveTypes`
Master list of leave categories. Seeded by `LeaveService.seedLeaveTypes()`.

```
leaveTypes/
  {docId}/
    name:           string      ← "Casual Leave", "Sick Leave", etc.
    maxDaysPerYear: number
    isPaid:         boolean
    isActive:       boolean
```

**Default documents seeded:**
| Name           | Max Days | Paid  |
|----------------|----------|-------|
| Casual Leave   | 12       | true  |
| Sick Leave     | 6        | true  |
| Earned Leave   | 15       | true  |
| Unpaid Leave   | 365      | false |

---

### `leaveRequests`
One document per leave application.

```
leaveRequests/
  {docId}/
    userId:        string       ← UID of applicant
    userName:      string
    leaveTypeId:   string       ← ID from leaveTypes
    leaveTypeName: string
    fromDate:      string       ← YYYY-MM-DD
    toDate:        string       ← YYYY-MM-DD
    totalDays:     number
    reason:        string
    status:        string       ← "pending" | "approved" | "rejected"
    approvedBy:    string?      ← UID of admin
    approvedOn:    timestamp?
    comments:      string?
    createdAt:     timestamp
```

---

### `tasks`
Task assignments from admin/manager to employees.

```
tasks/
  {docId}/
    title:           string
    description:     string
    assignedTo:      string     ← UID
    assignedToName:  string
    assignedBy:      string     ← UID
    assignedByName:  string
    status:          string     ← "pending" | "in_progress" | "completed" | "on_hold"
    priority:        string     ← "low" | "medium" | "high"
    dueDate:         string     ← YYYY-MM-DD
    createdAt:       timestamp
    updatedAt:       timestamp

  {docId}/workdone/             ← subcollection
    {logId}/
      userId:    string
      note:      string
      hours:     number
      loggedAt:  timestamp
```

---

### `hrRules`
Configurable policy rules. Document ID = rule name. Seeded by `HrRulesService.seedDefaults()`.

```
hrRules/
  office_intime/
    value:       "09:30"
    description: "Standard office in-time (HH:MM)"
    category:    "attendance"

  office_outtime/
    value:       "18:30"
    description: "Standard office out-time (HH:MM)"
    category:    "attendance"

  late_threshold_minutes/
    value:       "15"
    description: "Grace period in minutes before marking late"
    category:    "attendance"

  max_late_marks_per_month/
    value:       "3"
    description: "Max late marks allowed per month"
    category:    "attendance"

  halfday_threshold_hours/
    value:       "4"
    description: "Hours below which attendance is counted as half-day"
    category:    "attendance"
```

---

### `notifications`
In-app alerts for users.

```
notifications/
  {docId}/
    userId:     string          ← UID of recipient
    title:      string
    message:    string
    type:       string          ← "leave" | "task" | "attendance" | "general"
    isRead:     boolean
    relatedId:  string?         ← ID of related leave/task document
    createdAt:  timestamp
```

## Login Flow — Username Authentication

Firebase Auth does not support username-only login natively, but we use a simple trick:

> The app silently converts the username to a fake email before sending it to Firebase.

```
user types:   john
app converts: john@hrapp.internal
Firebase sees: john@hrapp.internal  ← user never knows this
```

`hrapp.internal` is not a real domain — Firebase just treats it as a valid email string. The user only ever types their **username** and **password** in the login screen.

### How it works in code (`auth_repository.dart`):
```dart
final email = '${username.trim().toLowerCase()}@hrapp.internal';
await _auth.signInWithEmailAndPassword(email: email, password: password);
```

### What you must do in Firebase when creating any user:

When adding a user in **Firebase Console → Authentication → Add user**, always set the email as:
```
username@hrapp.internal
```

For example:
| Username | Firebase Auth Email         |
|----------|-----------------------------|
| `admin`  | `admin@hrapp.internal`      |
| `john`   | `john@hrapp.internal`       |
| `sarah`  | `sarah@hrapp.internal`      |

> ⚠️ If you set the email to anything else (e.g. a real email), that user will **not** be able to log in via the app's username field.

### Role-based routing after login:
- `role == "admin"` → **Admin Dashboard**
- `role == "employee"` → **Employee Dashboard**

## Firestore Security Rules (for production)

Once development is done, replace **test mode** rules with:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can read their own profile; admins can read all
    match /users/{uid} {
      allow read: if request.auth.uid == uid || isAdmin();
      allow write: if isAdmin();
    }

    // Attendance: employee reads/writes own; admin reads all
    match /attendance/{doc} {
      allow read, write: if request.auth.uid == resource.data.userId || isAdmin();
      allow create: if request.auth != null;
    }

    // Leave requests: employee manages own; admin manages all
    match /leaveRequests/{doc} {
      allow read, write: if request.auth.uid == resource.data.userId || isAdmin();
      allow create: if request.auth != null;
    }

    // Tasks: assigned user or admin
    match /tasks/{doc} {
      allow read, write: if request.auth.uid == resource.data.assignedTo || isAdmin();
    }

    match /tasks/{doc}/workdone/{log} {
      allow read, write: if request.auth != null;
    }

    // Notifications: only the recipient
    match /notifications/{doc} {
      allow read, write: if request.auth.uid == resource.data.userId;
    }

    // HR Rules and Leave Types: read by all, write by admin only
    match /hrRules/{doc} {
      allow read: if request.auth != null;
      allow write: if isAdmin();
    }
    match /leaveTypes/{doc} {
      allow read: if request.auth != null;
      allow write: if isAdmin();
    }

    function isAdmin() {
      return get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
  }
}
```
